# Cortex

> Your company's knowledge base — queryable in plain language.

Cortex is a multi-tenant knowledge base platform. Organizations host their entire
document library on Cortex, and employees ask questions in natural language.
Answers are generated by a RAG (retrieval-augmented generation) pipeline grounded
in the company's own documents — not the open internet.

---

## How It Works

```text
Employee asks a question
       ↓
Cortex searches the workspace's document library for relevant passages
       ↓
Passages are passed to the LLM as context
       ↓
LLM generates an answer grounded in company knowledge
       ↓
Answer is returned with source citations
```

Each organization (workspace) has its own isolated document library, users, and
roles. One deployment of Cortex can serve many organizations simultaneously.

## Features

### Workspaces (Multi-tenancy)

- Each organization gets an isolated workspace
- Documents, users, and roles are scoped per workspace
- No data leakage between organizations

### Knowledge Base

- Upload PDF documents into the workspace library
- Documents are chunked and embedded automatically
- Supports large document collections

### Natural Language Q&A

- Employees ask questions in plain language
- RAG retrieves the most relevant document passages
- LLM generates answers grounded strictly in workspace documents
- Every answer includes source citations with relevance scores
- Configurable relevance threshold and number of sources

### Access Control

| Role       | Capabilities                                              |
| ---------- | --------------------------------------------------------- |
| **Admin**  | Manage workspace, users, roles, and all documents         |
| **User**   | Upload and manage own documents, query the knowledge base |
| **Public** | Browse documents and query the knowledge base (read-only) |

## Setup

### Backend

```bash
cd backend
cp .env.example .env   # fill in required values
uv sync
uv run python main.py
```

API runs at `http://localhost:8000`. Docs at `http://localhost:8000/docs`.

**Environment variables:**

```env
JWT_SECRET_KEY=replace-with-a-long-random-secret-at-least-32-characters
ACCESS_TOKEN_EXPIRE_MINUTES=60
DATABASE_URL=sqlite:///./app.db
CORS_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
EMBEDDINGS_API_KEY=
GITHUB_TOKEN=
```

- `JWT_SECRET_KEY` — required, minimum 32 characters.
- `CORS_ORIGINS` — comma-separated list of allowed frontend origins.
- `EMBEDDINGS_API_KEY` — API key for the embeddings provider used in the RAG pipeline.

**Create the first admin:**

```bash
uv run python main.py create-admin --name "Admin" --email admin@example.com --password secret
```

### Frontend

```bash
cd frontend
npm install
npm run dev
```

Frontend runs at `http://localhost:5173`.

**Environment variables:**

```env
VITE_API_URL=http://localhost:8000
```
